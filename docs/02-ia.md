# 🗺️ Information Architecture (IA)

## 🎯 핵심 UX 원칙

**카훗 스타일 미니멀리즘**

1. **⚡ 빠른 시작**: 3단계로 게임 시작 (선택 → 확인 → 플레이)
2. **🎯 명확한 액션**: "방 생성하기" 버튼 하나만
3. **⭐ 즐겨찾기 우선**: 별표한 게임이 맨 앞에
4. **✏️ 편집 필수**: 내용 확인 후 방 생성 (실수 방지)
5. **2개 탭만**: 둘러보기 / 내 게임

---

## 📱 화면 구조

> **🎯 Kahoot 스타일 역할 분리**: 주최자(Host)와 참가자(Participant) 화면을 명확하게 구분하여 각 역할에 최적화된 경험 제공

---

## 🎮 주최자(Host) 화면 구조

> **주최자 역할**: 게임 생성, 관리, 진행, 통계 확인

**핵심 기능:**

**좌측 - 질문 목록 (Question List)**:
- **[+ 질문 추가]**: 새 질문 추가 (자동으로 편집 패널에 표시)
- **질문 카드 클릭**: 해당 질문이 중앙 편집 패널에 로드됨
- **⏱️ 시간 표시**: 각 질문의 시간 제한 표시
- **[🗑️ 삭제]**: 질문 삭제 (확인 없이 즉시)
- **드래그 앤 드롭**: 질문 순서 변경 (예정)

**중앙 - 편집 패널 (Edit Panel)**:
- **질문 텍스트**: 실시간 입력 (우측 미리보기에 즉시 반영)
- **질문 유형 선택**: 객관식 / OX 퀴즈 / 단답형 / 밸런스 게임
- **선택지 입력**: 질문 유형에 따라 동적으로 변경
- **이미지 URL**: 선택적 이미지 추가 (미리보기에 표시)
- **시간 제한**: 질문별 개별 설정 (10초~120초)
- **[저장]**: 편집 내용 저장 (자동 저장도 지원)

**우측 - 미리보기 (Preview)**:
- **실시간 렌더링**: 편집 내용이 즉시 반영
- **실제 게임 화면**: 참가자가 보는 화면 그대로 표시
- **타이머 표시**: 설정된 시간 제한 확인

**상단 툴바**:
- **☆ 즐겨찾기**: 우측 상단에서 즐겨찾기 토글
- **[⚙️ 게임 설정]**: 전체 게임 설정 (효과음, 테마 등)
- **[⚡ 일괄 설정]**: 여러 질문의 시간 제한 한 번에 설정
- **[저장하고 방 생성]**: 저장 + PIN 발급 화면으로 이동

**UX 개선점 (Modal → Side Panel)**:

| Before (Modal) | After (3-Column) |
|---------------|-----------------|
| ❌ 질문 목록이 모달에 가려짐 | ✅ 항상 좌측에 표시 |
| ❌ 모달 열기/닫기 반복 | ✅ 클릭 한 번으로 전환 |
| ❌ 미리보기 없음 | ✅ 실시간 미리보기 |
| ❌ 다음 질문 보려면 모달 닫아야 함 | ✅ 목록에서 바로 선택 |
| ⏱️ ~10초/질문 | ⏱️ ~3초/질문 (3배 빠름) |

**플로우:**

- **둘러보기에서 온 경우**:
  - 템플릿 내용 확인/수정
  - [저장하고 방 생성] → 자동으로 "내 게임"에 저장 → PIN 발급
- **내 게임에서 [✏️ 편집] 클릭**:
  - 수정만 하고 저장
  - 방 생성 안 함

- **[방 생성하기]**: 편집 없이 바로 PIN 발급 (이미 내가 만든 게임)
- **[✏️ 편집]**: 편집 화면으로 이동 (수정만, 방 생성 안 함)
- **[🗑️ 삭제]**: 확인 모달 후 삭제
- **필터**: 즐겨찾기만 / 최근 플레이순 / 이름순

**"내 게임"에 저장되는 경우:**

1. 둘러보기에서 ⭐ 클릭 → 복사본 생성
2. 둘러보기에서 [방 생성하기] → 편집 → [저장하고 방 생성] → 복사본 생성
3. 내 게임에서 [방 생성하기] → 플레이 횟수만 증가

## 🎨 반응형 디자인

### Desktop (주최자 전용)

- **게임 진행 시**: 전체화면 프레젠테이션 모드 (프로젝터 연결)
- **게임 관리**: 카드 그리드 레이아웃 (3-4열)
- **편집 화면**: 넓은 편집 공간, 드래그 앤 드롭 지원
- **통계 및 리더보드**: 상세 정보 표시

### Mobile (참가자 최적화)

- **큰 버튼**: 터치 최적화 (최소 44x44px)
- **심플한 UI**: PIN 입력, 선택 버튼만 (카훗 스타일)
- **세로 모드**: 세로 모드 고정 (가로 회전 불필요)
- **실시간 동기화**: 주최자 화면과 자동 동기화
- **카드 리스트**: 1-2열 레이아웃

### Tablet (주최자/참가자 공용)

- **주최자**: 태블릿으로 게임 진행 가능 (프레젠테이션 모드)
- **참가자**: 모바일과 동일한 경험
- **카드 그리드**: 2-3열 레이아웃
- **가로/세로**: 모두 지원 (주최자는 가로 권장)

```

     → 미리보기 → 커스터마이징 → 방 생성 → 게임 시작
```

### After (개선 IA - 간단!)

```
로그인 → 게임 선택 → 편집 확인 → 방 생성 → 게임 시작
(4단계! ⚡)
```

### 주요 개선 사항

1. **✅ "대시보드" 제거**: 로그인 후 바로 "둘러보기"
2. **✅ "미리보기" 선택사항**: [👁️ 미리보기] 버튼만
3. **✅ 편집 화면 필수**: 내용 확인 후 방 생성 (실수 방지)
4. **✅ 즐겨찾기 우선**: ⭐ 게임이 맨 위에
5. **✅ 2개 탭만**: 둘러보기 / 내 게임
6. **✅ 1개 버튼**: "방 생성하기" (명확)

## 🎯 Kahoot 스타일 역할 분리 요약

### 주최자(Host) vs 참가자(Participant) 비교

| 구분 | 주최자(Host) | 참가자(Participant) |
|------|-------------|-------------------|
| **주요 화면** | 게임 생성, 관리, 진행 | 게임 입장, 플레이 |
| **디바이스** | 데스크톱/태블릿 (프레젠테이션) | 모바일 (터치 최적화) |
| **기능** | 게임 설정, 통계, 리더보드 | 답변 제출, 결과 확인 |
| **화면 수** | 10개 화면 (H1-H10) | 7개 화면 (P1-P7) |
| **플로우 복잡도** | 중간 (게임 관리 필요) | 매우 간단 (PIN 입력만) |

### Kahoot에서 참고한 핵심 구조

1. **명확한 역할 분리**
   - 주최자: 게임 생성 및 진행 관리 (데스크톱 중심)
   - 참가자: 간단한 입장 및 플레이 (모바일 중심)

2. **주최자 화면 특징** (Kahoot 스타일)
   - 질문과 선택지를 대형 화면에 표시
   - 실시간 응답 현황 및 통계 표시
   - 리더보드 및 참가자 관리 기능
   - 게임 진행 제어 (다음 질문, 정답 공개 등)

3. **참가자 화면 특징** (Kahoot 스타일)
   - PIN 입력으로 즉시 입장
   - 자신의 기기에서 질문과 선택지 확인 가능
   - 큰 버튼으로 답변 제출 (터치 최적화)
   - 실시간 순위 및 점수 확인

4. **화면 동기화**
   - 주최자 화면과 참가자 화면이 실시간으로 동기화
   - 주최자가 질문을 진행하면 참가자 화면 자동 업데이트
   - WebSocket을 통한 실시간 통신

5. **프레젠테이션 모드**
   - 주최자는 전체화면 모드로 프로젝터 연결 가능
   - 참가자는 자신의 기기에서 독립적으로 플레이
