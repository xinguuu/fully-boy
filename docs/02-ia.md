# 🗺️ Information Architecture (IA)

## 🎯 핵심 UX 원칙

**카훗 스타일 미니멀리즘**

1. **⚡ 빠른 시작**: 3단계로 게임 시작 (선택 → 확인 → 플레이)
2. **🎯 명확한 액션**: "방 생성하기" 버튼 하나만
3. **⭐ 즐겨찾기 우선**: 별표한 게임이 맨 앞에
4. **✏️ 편집 필수**: 내용 확인 후 방 생성 (실수 방지)
5. **2개 탭만**: 둘러보기 / 내 게임

**플로우:**

- **둘러보기에서 온 경우**:

  - [저장하고 방 생성] → 자동으로 "내 게임"에 저장 → PIN 발급
- **내 게임에서 [✏️ 편집] 클릭**:
  - 수정만 하고 저장
  - 방 생성 안 함

**핵심 기능:**

- **⭐ 섹션 분리**: 즐겨찾기가 맨 위에
- **[방 생성하기]**: 편집 없이 바로 PIN 발급 (이미 내가 만든 게임)
- **[✏️ 편집]**: 편집 화면으로 이동 (수정만, 방 생성 안 함)
- **[🗑️ 삭제]**: 확인 모달 후 삭제
- **필터**: 즐겨찾기만 / 최근 플레이순 / 이름순

**"내 게임"에 저장되는 경우:**

1. 둘러보기에서 ⭐ 클릭 → 복사본 생성
2. 둘러보기에서 [방 생성하기] → 편집 → [저장하고 방 생성] → 복사본 생성
3. 내 게임에서 [방 생성하기] → 플레이 횟수만 증가

### B. 진행자(관리자) 플로우

#### 방법 1: 빠른 플레이 (⚡ 추천)

```
1. 로그인 → 둘러보기 (자동 이동)
   ↓
2. 게임 선택 (밸런스 게임)
   ↓
3. [방 생성하기] 클릭
   ↓
4. 편집 화면 (내용 확인/수정)
   ↓
5. [저장하고 방 생성] 클릭
   ↓
6. PIN 발급 (123456)
   ↓
7. 참가자 입장 대기
   ↓
8. [게임 시작] → 게임 진행 → 결과
```

**소요 시간: 5분 이내**

#### 방법 2: 즐겨찾기 후 나중에 플레이

```
1. 둘러보기 → 게임 선택
   ↓
2. ⭐ 클릭 (즐겨찾기 추가)
   ↓
3. "내 게임"에 자동 저장
   ↓
4. 나중에 "내 게임" → [방 생성하기]
   ↓
5. PIN 발급 → 게임 시작
```

#### 방법 3: 내 게임 편집만

```
1. 내 게임 → 게임 선택
   ↓
2. [✏️ 편집] 클릭
   ↓
3. 편집 화면 → 수정
   ↓
4. [저장] 클릭
   ↓

```

## 🎨 반응형 디자인

### Desktop (관리자)

- 게임 진행 시 전체화면 (프레젠테이션 모드)
- 카드 그리드 레이아웃 (3-4열)
- 넓은 편집 공간

### Mobile (참여자)

- 큰 버튼 (터치 최적화)
- 심플한 UI (PIN 입력, 선택 버튼)
- 세로 모드 최적화
- 카드 리스트 레이아웃 (1-2열)

### Tablet (관리자/참여자)

- 관리자가 태블릿으로 진행 가능
- 카드 그리드 레이아웃 (2-3열)
- 가로/세로 모두 지원

## 💡 핵심 UX 개선점

### Before (기존 IA - 복잡)

```
로그인 → 대시보드 → 게임 라이브러리 → 게임 선택
     → 미리보기 → 커스터마이징 → 방 생성 → 게임 시작
(8단계!)
```

### After (개선 IA - 간단!)

```
로그인 → 게임 선택 → 편집 확인 → 방 생성 → 게임 시작
(4단계! ⚡)
```

### 주요 개선 사항

1. **✅ "대시보드" 제거**: 로그인 후 바로 "둘러보기"
2. **✅ "미리보기" 선택사항**: [👁️ 미리보기] 버튼만
3. **✅ 편집 화면 필수**: 내용 확인 후 방 생성 (실수 방지)
4. **✅ 즐겨찾기 우선**: ⭐ 게임이 맨 위에
5. **✅ 2개 탭만**: 둘러보기 / 내 게임
6. **✅ 1개 버튼**: "방 생성하기" (명확)
