# 🎮 Xingu - 한국형 파티 게임 플랫폼

## 📌 서비스 개요

**Xingu**는 오프라인 파티, MT, 워크샵에서 쉽고 재미있게 즐길 수 있는 **한국형 파티 게임 플랫폼**입니다.

### 💡 핵심 컨셉
> **"카훗의 편리함 + 한국 예능/밈 게임 콘텐츠"**

**카훗(Kahoot)의 플로우**를 기반으로:
- 진행자가 게임 선택 → 방 생성 → 참여자 PIN 입장
- 실시간 인터랙션, 순위 경쟁, 결과 시각화

**+ 한국 특화 콘텐츠**:
- 예능(지구오락실, 런닝맨) 게임 템플릿
- SNS 유행 밈 게임
- 초성퀴즈, 밸런스게임 등 한국 문화

**+ 유연한 진행 방식**:
- **모바일 필요**: 참여자가 폰으로 참여 (카훗 방식)
- **모바일 불필요**: 진행자만 조작, 참여자는 오프라인 참여

### 🎯 게임 프레임워크 제공
진행자는 **게임 틀**을 선택하고, **내용만 수정**하면 5분 만에 준비 완료!

**예시**:
- "밸런스 게임" 템플릿 선택 → 질문 5개만 수정 → 완료
- "초성 퀴즈" 템플릿 선택 → 단어 10개만 입력 → 완료
- "OX 퀴즈" 템플릿 선택 → 문제 수정 → 완료

## 🎯 핵심 가치

### 1. ⚡ 빠른 준비 (5분 컷!)
> "예능에서 본 게임, 바로 우리 상황에 맞게!"

- 게임 **프레임워크만 제공**, 내용은 수정만 하면 끝
- 처음부터 만들 필요 없음 (Kahoot은 30분, Xingu는 5분)
- 지구오락실, 런닝맨 등 예능 게임 → 즉시 따라 할 수 있게

### 2. 🔥 트렌디한 콘텐츠
> "지금 유행하는 게임, 여기 다 있어요"

- SNS/예능에서 유행하는 게임 **매주 업데이트**
- 한국 밈/문화에 최적화 (초성퀴즈, 밸런스게임, 케미테스트)
- 커뮤니티에서 인기 게임 공유

### 3. 🎮 유연한 진행 방식
> "상황에 맞게 선택"

- **MC 모드**: 폰 없이 진행 (밸런스게임, 초성퀴즈)
- **모바일 모드**: 카훗처럼 실시간 경쟁 (퀴즈, 투표)

### 4. ✏️ 쉬운 커스터마이징
> "우리 상황에 딱 맞게"

- 질문/내용만 수정하면 완성
- 드래그 앤 드롭으로 순서 변경
- 이미지/음원/영상 자유롭게 추가

## 👤 사용자 플로우

### 🎯 진행자(관리자) 플로우 - 핵심!

```
1. 로그인/회원가입
   ↓
2. 게임 라이브러리 탐색
   - 📂 카테고리 필터: 아이스브레이킹, 퀴즈, 예능게임, 밈게임
   - 📱 모바일 필요 여부: "모바일 필요" / "모바일 불필요"
   - ⏱️ 시간: 5분, 10분, 30분
   - 👥 인원: 소규모, 중규모, 대규모
   ↓
3. 게임 선택 & 미리보기
   - 게임 설명 확인
   - 예시 화면 확인
   ↓
4. 게임 커스터마이징 (선택)
   - 질문/내용 수정
   - 이미지/음원 추가
   - 시간/점수 설정
   - "내 템플릿"으로 저장
   ↓
5. 방 생성
   - 게임 제목 입력
   - PIN 코드 자동 발급 (6자리)
   - QR 코드 생성
   ↓
6. 참여자 대기
   - 참여자 입장 확인 (실시간)
   - 준비 완료 버튼
   ↓
7. 게임 시작 & 진행
   - [모바일 불필요]: 진행자가 화면 조작, 참여자는 오프라인 참여
   - [모바일 필요]: 참여자가 폰으로 선택/투표
   ↓
8. 결과 확인
   - 순위 발표
   - 통계 확인
   - 결과 공유/저장
```

### 📱 참여자 플로우 (카훗 방식)

```
1. Xingu 접속 (웹 브라우저)
   ↓
2. PIN 코드 입력
   - 진행자가 공유한 6자리 PIN
   - 또는 QR 코드 스캔
   ↓
3. 닉네임 설정
   - 게임에서 표시될 이름 입력
   ↓
4. 대기실
   - 다른 참여자 입장 확인
   - 게임 시작 대기
   ↓
5. 게임 참여
   - [모바일 불필요 게임]: 진행자 화면만 보면서 오프라인 참여
   - [모바일 필요 게임]: 폰 화면에서 선택/투표 버튼 클릭
   ↓
6. 결과 확인
   - 최종 순위 확인
   - 점수 확인
```

### 2. 게임 커스터마이징 (핵심!)
> **게임 "틀"은 그대로, 내용만 쉽게 수정**

#### 📝 콘텐츠 편집
- **질문/내용 수정**: 텍스트 박스에서 바로 수정
- **항목 추가/삭제**: 질문 개수 자유롭게 조정

- **효과음/배경음**: ON/OFF

## 🚀 Quick Start

### Prerequisites

- **Node.js**: 24+
- **Docker**: Docker Desktop or Docker Engine + Docker Compose
- **PostgreSQL**: 17+ (or use Docker)
- **Redis**: Latest (or use Docker)
- **Git**: For version control

### Getting Started

#### Option 1: Local Development (Without Docker)

```bash
# Clone repository
git clone https://github.com/your-org/xingu.git
cd xingu

# Install dependencies (Turborepo will install all workspace packages)
npm install

# Set up environment variables
cp .env.example .env

# Run database migrations
npm run db:migrate

# Start all services in development mode
npm run dev

# Access the application
# - Frontend: http://localhost:3000
# - Auth Service: http://localhost:3001
# - Game Service: http://localhost:3002
# - WS Service: http://localhost:3003
```

#### Option 2: Docker Development

```bash
# Clone repository
git clone https://github.com/your-org/xingu.git
cd xingu

# Start all containers (first time will build images)
docker-compose up

# Or build and start
docker-compose up --build

# Access the application
# - Frontend: http://localhost (via Nginx)
# - All services are accessible through Nginx reverse proxy

# Stop all containers
docker-compose down

# View logs
docker-compose logs -f web
docker-compose logs -f auth-service
```

#### Useful Commands

```bash
# Run specific service only
npm run dev --filter=web
npm run dev --filter=auth-service

# Run tests
npm run test
npm run test:e2e

# Build for production
npm run build

# Type checking
npm run type-check

# Linting
npm run lint
```

## 📚 Technology Stack

### 🏗️ Architecture

- **Pattern**: Microservice Architecture (MSA)
- **Monorepo**: Turborepo (npm workspaces)
- **Containerization**: Docker + Docker Compose
- **Total Containers**: 7 containers

### 🌐 API Architecture

- **REST API**: Auth Service, Game Service (RESTful endpoints)
  - Auth Service: `/api/auth/*` (signup, login, logout, refresh)
  - Game Service: `/api/games/*` (CRUD games, templates, rooms)
- **WebSocket**: WS Service (Socket.io for real-time communication)
  - Real-time events: room join/leave, game state, player actions
  - Pub/Sub via Redis for horizontal scaling
- **API Gateway**: Nginx (reverse proxy, load balancing, rate limiting)
  - Single entry point for all services
  - SSL/TLS termination
  - Static file serving

### 📦 Container Overview
1. **nginx** - Reverse proxy, load balancing
2. **postgres** - Main database (users, games, rooms)
3. **redis** - Session sharing, real-time state
4. **web** - Next.js frontend (SSR)
5. **auth-service** - NestJS (authentication, users)
6. **game-service** - Express (games, rooms, templates)
7. **ws-service** - Socket.io (real-time WebSocket)

### 🌐 Frontend (Next.js 15)

## 🔒 Security Strategy (MVP)

### Authentication & Authorization

- **JWT Access Token**: 15 minutes expiry (short-lived)
- **Refresh Token**: 7 days expiry (stored in Redis)
- **Session Store**: Redis (shared across all services)
- **Password Hashing**: bcrypt (salt rounds: 10)

### API Security

- **CORS Configuration**: Nginx + service-level CORS
  - Whitelist allowed origins
  - Credentials support for cookies
- **Rate Limiting**: Redis + Nginx
  - Auth endpoints: 5 requests/minute per IP
  - API endpoints: 100 requests/minute per user
  - WebSocket connections: 10 connections per user
- **Input Validation**: Zod schemas (shared across services)
  - Request body validation
  - Query parameter validation
  - Type-safe validation

### Data Security

- **SQL Injection Prevention**: Prisma ORM (parameterized queries)
- **XSS Prevention**: React (auto-escaping) + Content Security Policy
- **CSRF Protection**: SameSite cookies + CSRF tokens
- **Environment Variables**: Sensitive data in `.env` (not committed)
- **Secrets Management**: Docker secrets (production)

### Infrastructure Security

- **HTTPS Only**: SSL/TLS via Nginx (Let's Encrypt in production)
- **Security Headers**: Nginx configuration
  - `X-Frame-Options: DENY`
  - `X-Content-Type-Options: nosniff`
  - `X-XSS-Protection: 1; mode=block`
  - `Strict-Transport-Security: max-age=31536000`
- **Docker Network Isolation**: Internal network for services
- **Database Access**: Only services can access PostgreSQL (not exposed externally)

### Logging & Monitoring (Future)

- **Error Tracking**: Sentry (client + server)
- **Audit Logs**: User actions, authentication events
- **Security Monitoring**: Failed login attempts, suspicious activities

## 🎯 차별화 포인트

| 기능 | Kahoot | Xingu |
|-----|--------|-------|
| **게임 유형** | 퀴즈 중심 | 퀴즈 + 예능 게임 + 유행 밈 게임 |
| **콘텐츠** | 직접 만들어야 함 | **프레임워크 제공** (틀만 가져다 쓰면 됨) |
| **참여 방식** | 모바일 필수 | MC 모드 (폰 불필요) + 모바일 모드 |
| **타겟** | 교육/기업 학습 | 파티/MT/친목 (재미 중심) |
| **언어/문화** | 영어 중심, 글로벌 | 한국어 + 한국 예능/밈 문화 |
| **커스터마이징** | 제한적 | 자유로운 수정 (내용만 바꾸면 끝) |
| **게임 준비** | 처음부터 만들어야 함 | 템플릿 복사 → 내용만 수정 (5분 완성) |

### 🌟 핵심 차별점: "게임 프레임워크"

**Kahoot**: 
```
빈 퀴즈 → 질문 10개 작성 → 정답 입력 → 이미지 찾기 → 시간 설정
= 30분 소요
```

**Xingu**:
```
"밸런스 게임" 템플릿 선택 → 질문 5개만 수정 → 완료
= 5분 소요 ⚡
```

**예시**:
- 지구오락실 "이상한 단어 찾기" 게임 보고 → Xingu에서 해당 틀 선택 → 단어만 바꿔서 우리 상황에 맞게 사용
- 틱톡에서 유행하는 "찐친 테스트" → Xingu에서 템플릿 제공 → 우리 친구 이름/상황만 입력
